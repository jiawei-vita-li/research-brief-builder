## English

### Who I am

I am a graduate student with a Computer Science and Biomedical Engineering background. I am currently exploring a career direction that combines AI agents, AI-native product design, developer tools, and possible PhD research.

My current resume includes:
- A five-layer LLM agent project with Perception / Planner / Executor / Evaluator / Reflection modules.
- ReAct / Reflexion-style tool use.
- MCP-style tool schema design.
- Replayable trajectory logging.
- LLM-as-Judge evaluation.
- Multimodal AI and foundation model adaptation.
- FastAPI / Dify / Agent-callable generation tool experience.

### Why I am building this project

I am preparing for AI product internships, especially AI coding / developer tool / agent workflow roles such as TRAE.

I also have not given up the PhD route, so this product should support both:
1. AI product portfolio building.
2. Real RA outreach and paper-reading workflow.

### My product positioning

I do not want this to be a generic chatbot or a simple paper summarizer.

I want this to be an AI-native workflow product:
- visible steps,
- structured intermediate outputs,
- editable AI-generated artifacts,
- human-in-the-loop review,
- traceable reasoning,
- exportable final deliverables.

### What I care about

When implementing this project, prioritize:
- product clarity,
- workflow visibility,
- clean UI,
- structured LLM outputs,
- realistic MVP scope,
- demo quality for AI product interviews,
- usefulness for real RA outreach.

Avoid:
- overengineering,
- broad scraping features in the MVP,
- vague chatbot UX,
- hallucinated paper metadata,
- unnecessary backend complexity.

## 中文

### 我是谁

研究生，计算机 + 生物医学工程背景。现在在找一条能把AI Agent、开发者工具、以及可能的博士方向串起来的路。

简历里比较拿得出手的有：
- 五层LLM Agent：Perception / Planner / Executor / Evaluator / Reflection。
- ReAct / Reflexion 式工具调用。
- MCP风格工具schema设计。
- 可回放轨迹日志。
- LLM-as-Judge评估。
- 多模态与基础模型适配。
- FastAPI / Dify / Agent可调用的生成工具。

### 为什么做这个项目

在冲AI产品实习，偏AI编程、开发者工具、Agent工作流（比如TRAE）。

读博也没完全放下，所以这个东西要两头顾：
1. 作品集里能讲清楚产品思路。
2. 自己真用来读论文、做RA套磁。

### 产品定位

不要做成「又一个聊天机器人」，也不要做成「粘贴就摘要」。

要的是**按步骤走的工作流**：
- 每一步看得见，
- 中间结果是结构化的，
- AI写出来的东西你能改，
- 关键步骤你来把关，
- 推理过程能追溯，
- 最后能导出带走。

### 实施时的优先级

写代码时优先：
- 产品讲清楚，
- 流程别藏，
- UI干净好用，
- LLM输出有schema、能解析，
- MVP范围别贪，
- 面试演示要顺，
- 套磁时真能省时间。

别碰：
- 过度工程，
- MVP里上大爬取，
- 聊天式糊一片的UX，
- 论文元数据瞎编，
- 没必要的服务端复杂度。
